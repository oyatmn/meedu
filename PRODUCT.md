# Product

## Register

product

## Users

**主要用户：网校运营人员**
长时间在桌面环境（多窗口、并行任务）使用 admin 后台，需要快速处理课程编排、订单、学员、内容审核与异常排查。属于熟练用户，能容忍信息密度，但不接受信息歧义和无谓的视觉干扰。

**次要用户：付费学员（pc / h5）**
以职业培训学员与兴趣付费用户为主，目标是顺畅地完成"找课 → 购买 → 学习 → 复看"的闭环。使用场景碎片化，对加载速度、视频播放稳定性、登录与支付路径敏感。学习过程中的心流体验是核心资产。

## Product Purpose

MeEdu 是数据自托管的开源网校 / 知识付费系统。它要让中小型培训机构、个体讲师、企业内训团队，能在不依赖第三方平台的前提下，独立完成课程售卖、学员管理与播放分发。

成功的定义：

- 运营侧能用最少的点击完成日常运营动作，看得清现状，定位得到异常
- 学员侧不会因为产品的体验问题而流失，学习路径短、播放稳、支付顺
- 开源版本本身可用、可部署、可读，不靠"残缺感"去逼用户升级商业版

## Brand Personality

**三词总结：可靠、专业、克制**

- **可靠**：交易系统、播放系统、权限系统是底盘——视觉与交互必须传递"这套东西能撑住生产"的信号，而不是 demo 感
- **专业**：默认是 B 端工具的语气，不卖萌、不堆 emoji、不喧哗；面向运营和讲师时用行业词汇，面向学员时用清晰的人话
- **克制**：视觉上不靠强色、强渐变、强装饰来抓人——靠结构、留白与字重层次说话。开源版的克制还体现在不用 UI 暗示"功能阉割"

正面对标：腾讯课堂——B 端 + 学员端混合形态、信息层级清楚、不浮夸。

## Anti-references

明确不要做成的样子：

- **花哨的 K12 教育站**：卡通插画、彩虹渐变、奖章弹窗、放大的成就动画——与目标用户（成人付费学员、专业运营）不匹配
- **过度营销的微商课程页**：满屏倒计时、滚动小红字、"已有 XXX 人购买"的伪从众提示、价格红字加粗加闪烁——破坏可信度
- **老派的 OA 后台**：左侧深蓝竖条 + 顶部灰蓝渐变 + 默认 Element/Antd 配色不做任何主题——传递"工程师顺手搭的"信号，与"可靠的商业系统"目标背离
- **AI 风格化堆叠**：彩色玻璃卡片 + 大圆角 + 渐变文字 + 通用 hero 模板——任何让人一眼认出"这是 AI 套的模板"的元素

## Design Principles

1. **数据密度优先于留白**
   admin 是工作面板不是展示面。运营人员愿意为"一屏看完更多东西"付出认知成本。表格、统计、筛选区不靠加大间距来"显得高级"，靠对齐、字重、分组来分层。

2. **管理员能一眼定位异常**
   异常（订单失败、内容下架、配置缺失、超管未初始化、安全告警）必须在视觉层级里**自动浮上来**，不需要二次查找。颜色、徽标、空状态都为这条原则服务，而不是为"好看"服务。

3. **学员心流不被弹窗打断**
   pc / h5 端的播放、购买、登录路径里，禁止用 modal 兜底任何可以 inline 的状态。引导信息走非阻塞的形式（toast / sidebar / 顶部条），错误提示要在原位说清楚原因和下一步，而不是把人弹出去再让人找回来。

4. **开源版的视觉自信**（衍生自三词品牌）
   开源版本不靠"缺点"反向衬托商业版。免费功能本身要呈现完成态——配色、字号、空状态、错误页都按生产标准做。商业版价值靠功能本身体现，不靠让开源版看起来寒酸。

## Accessibility & Inclusion

无强制 WCAG 等级要求，也没有已知的特殊用户群体。但作为默认底线遵循：

- 文本与背景对比度按 WCAG AA（4.5:1 正文 / 3:1 大字号）执行
- 关键交互可用键盘完成（Tab/Enter/Esc 的语义遵循 Antd 默认）
- 颜色不作为信息的唯一载体（红色异常状态必须同时带图标或文字）
- 尊重系统的 `prefers-reduced-motion`，超出 200ms 的过渡需要可关
